## 回溯：不是盲目递归，而是搜索选择树

回溯题可以统一写成四个问题：

1. 当前路径 `path` 表示什么？
2. 这一层有哪些选择？
3. 到什么条件时记录答案？
4. 哪些选择已经违反约束，可以直接剪枝？

最基本的框架是：

```go
for _, choice := range choices {
    // 1. 做选择
    path = append(path, choice)

    // 2. 进入下一层
    dfs(nextState)

    // 3. 撤销选择，尝试兄弟分支
    path = path[:len(path)-1]
}
```

### 回文分割里的剪枝

131 题将字符串分成若干段，要求每一段都是回文。若当前决定切出 `s[start:i+1]`，这段内容已经固定；只有它是回文时，才可以放入 `path` 并递归处理后缀。

```go
substr := s[start : i+1]
if isPalindrome(substr) {
    path = append(path, substr)
    dfs(i+1, i+1)
    path = path[:len(path)-1]
}
```

例如 `"baa"` 不是回文，`["baa", ...]` 无论再怎样分割后面都不合法，因此无需沿着“把 `baa` 作为一段”的分支继续搜索。但算法仍会保留更早切分的分支，如 `["b", "aa"]`。

这说明剪枝不是停止整个程序，而是停止一个已经确定不可能成功的选择。

### 不同题目的状态设计

- 组合总和允许重复选同一个数，状态中需要保留剩余目标值；目标小于零时立即返回。
- 全排列的状态是“已选元素 + 未选元素”；每层取一个未使用元素。
- 子集的每个元素只有“选”或“不选”两种去向，本质是二叉选择树。
- 括号生成额外保持约束：右括号数量不能超过左括号数量。
- N 皇后和单词搜索则把路径放在棋盘上；每次落子或走格都要检查列、对角线或访问标记。

题目之间的差异主要来自状态，而不是递归骨架。先能说清状态，递归的参数和终止条件就会自然出现。

## 网格与图：访问策略决定题型

200 岛屿数量使用 DFS：遇到陆地就扩散访问相邻陆地，并把访问过的位置标记掉。一块连通陆地只会触发一次搜索。

994 腐烂的橘子更适合多源 BFS：第一分钟里所有已经腐烂的橘子共同作为起点，每次扩展一层就是过去一分钟。这里“队列当前长度”就是时间边界。

207 课程表先把先修关系转换为有向图。核心不是背模板，而是识别图中的环：若一门课程的 DFS 过程中再次遇到当前递归栈里的课程，就说明存在循环依赖，无法完成所有课程。

## Trie：把字符串比较变成沿边行走

208 题的 Trie 将每个节点看成一个前缀。插入字符串时按字符向下创建节点；搜索单词要求走到末尾节点且它被标记为完整单词；`StartsWith` 则只要求前缀路径存在。

相比把所有单词逐个比较，Trie 复用了公共前缀，适合大量前缀查询。

## 树路径题：先限定路径的起点和方向

437 路径总和 III 的路径可以从任意节点开始，但只能向下走。实现可以拆为两层：外层枚举每一个起点，内层从固定起点继续扣减目标值并向左右孩子搜索。虽然前缀和写法可以进一步优化，但这个拆分让“枚举起点”和“延长路径”各自职责清晰。

最大路径和练习则强调另一种 DFS 思维：每个节点向父节点只能贡献一条向下路径；经过当前节点的完整路径，可以组合左侧贡献、节点值和右侧贡献。关键在于区分“返回给父节点的值”和“更新全局答案的值”。

## 小结

这批代码的共同训练目标不是记住更多题解，而是形成稳定的提问顺序：

1. 我的状态包含哪些信息？
2. 每一层能做什么选择？
3. 哪些选择应立即剪掉？
4. 递归返回时，哪些现场必须恢复？

回溯的想象力，本质上是把题目翻译成选择树的能力；图搜索的重点是选对 DFS 或 BFS；而 Trie 和树递归都要求先定义好节点状态的含义。把这些模型练熟，新题就不再是一段完全陌生的代码。
